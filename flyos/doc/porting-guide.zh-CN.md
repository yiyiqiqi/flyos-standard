# flyOS 移植指南

[English](porting-guide.md)

本文档面向想在公开试用阶段评估或理解 flyOS 端口边界的开发者。

## 1. 当前公开端口

公开试用分支当前包含：

- `stm32f4`
- `s32k344`

它们的定位是公开试用目标，而不是对所有下游产品场景作出的完整能力承诺。

## 2. 移植层职责

`port/` 目录负责处理：

- 上下文切换；
- 中断与 Tick 相关底层支持；
- 平台相关底层接口；
- 与上层 kernel 的边界对接。

## 3. 上层工程需要准备什么

在尝试移植或集成前，上层工程通常需要已经具备：

- 启动文件与向量表；
- 链接脚本；
- 时钟初始化；
- 芯片厂商 HAL/SDK；
- 与当前端口设计匹配的底层硬件初始化能力。

## 4. 移植时先看哪里

建议按这个顺序阅读：

1. 根 `README.zh-CN.md`
2. `quick-start.zh-CN.md`
3. `configuration.zh-CN.md`
4. 当前目标平台 `port/<platform>/` 下的实现

## 5. 公开分支的现实边界

当前公开分支的目标是让外部开发者能够：

- 理解平台边界；
- 尝试构建与接入；
- 给出移植和试用反馈。

它不是一个已经承诺平台矩阵广泛覆盖、认证证据完备或长期 API 冻结的产品线。
