# 路线图

[English](ROADMAP.md)

本文档描述 `flyos-standard` 当前开发方向。

## 当前发布线：v0.1.x

重点：

- 让 kernel 更容易理解和评估；
- 保持 STM32F4 和 S32K344 的集成路径可用；
- 提高文档质量和反馈收集能力；
- 讲清平台边界和配置预期。

预期输出：

- 更清晰的上手路径；
- 更清楚的 API 与配置文档；
- 更可复现的 issue 报告方式；
- 第一轮用户反馈。

## 下一条发布线：v0.2.x

重点：

- 提升 kernel 的可用性和文档深度；
- 强化移植说明；
- 改进核心 IPC 与内存管理说明；
- 在维护成本可控的前提下，增加更结构化的示例。

## 长期方向

长期可能关注的方向包括：

- 更强的可靠性工程支撑；
- 更规范的验证与确认材料；
- 在维护成本合理时，扩展更多平台支持；
- 更清晰地定义子模块边界和版本预期；
- 在仓库稳定后，再考虑统一文档命名与布局。

## 这份路线图不承诺什么

它不承诺：

- 认证级证据；
- 冻结的长期 ABI/API 稳定性；
- 对所有子系统或未来方向的即时覆盖。
