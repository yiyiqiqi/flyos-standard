# 安全策略

[English](SECURITY.md)

## 适用范围

欢迎针对公开 `flyos-standard` 发布线提交与安全相关的报告，尤其包括以下方面：

- 公开内核模型中的权限或隔离失效；
- 通过公开 API 触发的内存破坏；
- 会导致不安全运行时行为的调度器或 IPC 缺陷；
- 破坏文档化内核假设的平台端口错误。

## 如何报告

请不要在公开 issue 中完整披露尚未修复的漏洞。

更合适的方式是通过项目约定的维护者联系方式私下沟通，并附带：

- 受影响的提交或版本；
- 目标平台；
- 影响概述；
- 复现步骤或 PoC；
- 问题是否影响调度、IPC、内存或端口行为。

## 响应预期

维护者应尽量做到：

1. 确认收到报告；
2. 评估影响和受影响范围；
3. 准备修复或缓解措施；
4. 在合适时进行协调披露。

## 重要边界说明

当前仓库并不宣称具备认证级保证能力。因此，相关报告应结合 `README.zh-CN.md`、`FAQ.zh-CN.md`、`ROADMAP.zh-CN.md` 中明确写出的当前行为和限制来评估。
