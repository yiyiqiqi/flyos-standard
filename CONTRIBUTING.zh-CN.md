# 参与 flyOS

[English](CONTRIBUTING.md)

感谢你考虑为 `flyos-standard` 提交贡献。

## 这个仓库当前在优化什么

当前仓库主要围绕以下目标展开：

- Cortex-M 平台上的试用体验；
- 清晰的内核学习与评估入口；
- 真实克制的能力边界说明；
- 结构化的外部反馈收集。

它当前并不适合承载大范围的猜想式功能扩展或缺少说明的子系统铺开。

## 提交贡献前建议先读

- `README.zh-CN.md`

## 欢迎的贡献类型

- 带清晰复现步骤的 bug 修复；
- 文档修正与可读性改进；
- 面向当前目标平台的移植修复；
- 对调度器、IPC、内存或平台边界的、小而可评审的改进。

## 贡献规则

- 改动保持小而清晰，便于评审。
- 行为或使用方式变化时同步更新文档。
- 不在公开文档中扩大安全或认证结论。
- 不把无关清理和功能改动混在一起。
- 保持当前分支聚焦 kernel 主线，不把它变成无关材料的堆放区。

## Pull Request 检查清单

- [ ] 本次改动有清晰的问题定义。
- [ ] 受影响的公开文档已在需要时同步更新。
- [ ] 目标平台或复现场景已说明。
- [ ] 本次改动没有引入不受支持的公开承诺。
- [ ] 本次改动仍处于公开 kernel 试用范围内。

## 安全问题

请不要在公开 issue 中披露尚未修复的安全敏感问题。请使用 `SECURITY.zh-CN.md` 中描述的私下报告流程。
